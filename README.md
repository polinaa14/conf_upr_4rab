Конвертер учебного конфигурационного языка в XML

**1. Общее описание**

Данный инструмент представляет собой парсер конфигурационного языка, который преобразует текстовые конфигурационные файлы в формат XML. Программа принимает входные данные на учебном конфигурационном языке через стандартный ввод и выводит результат в формате XML через стандартный вывод.

Особенности языка:
  а) Поддерживает числа, строки, массивы и словари
  
  б) Имеет возможность объявления констант
  
  в) Поддерживает вычисления константных выражений
  
  г) Включает функцию abs() и операцию сложения
  
  д) Поддерживает однострочные комментарии
  
# 2. Описание всех функций и настроек

## Класс `ConfigParser`

Основной класс, отвечающий за парсинг конфигурационных файлов.

### **Методы:**

#### **`__init__(self)`**
- **Назначение**: Инициализирует парсер
- **Что делает**: Создает словарь для хранения констант

#### **`parse(self, text)`**
- **Назначение**: Основной метод парсинга
- **Что делает**:
  - Удаляет комментарии (все, что после `\\`)
  - Извлекает и вычисляет константы (`let имя = значение;`)
  - Парсит основную структуру конфигурации
  - Возвращает XML-дерево

#### **`_evaluate_expression(self, expr)`**
- **Назначение**: Вычисляет константные выражения
- **Что делает**:
  - Обрабатывает выражения в вертикальных чертах `|...|`
  - Поддерживает функцию `abs()`
  - Поддерживает операцию сложения `+`

#### **`_parse_value(self, value_str)`**
- **Назначение**: Парсит отдельное значение
- **Что делает**:
  - Числа: `42`, `-100`
  - Строки: `q(Hello World)`
  - Массивы: `{1, 2, 3}`
  - Словари: `$[key: value]`
  - Константы: `|CONSTANT_NAME|`

#### **`_parse_array(self, content)`**
- **Назначение**: Парсит массивы значений
- **Что делает**: Разделяет элементы массива по запятым с учетом вложенности

#### **`_parse_dict(self, content)`**
- **Назначение**: Парсит словари (пары ключ-значение)
- **Что делает**: Разделяет пары ключ:значение по запятым с учетом вложенности

#### **`_parse_structure(self, text, parent)`**
- **Назначение**: Парсит основную структуру конфигурации
- **Что делает**: Обрабатывает словари и массивы верхнего уровня

#### **`_add_to_xml(self, data, parent)`**
- **Назначение**: Рекурсивно добавляет данные в XML-дерево
- **Что делает**:
  - Преобразует словари в XML-элементы
  - Преобразует массивы в списки элементов
  - Добавляет простые значения как текст


Для запуска файлов .conf    
```bash
Get-Content game.conf | python config_to_xml.py

Get-Content webserver.conf | python config_to_xml.py
```

## Пример работы программы

```xml
<?xml version="1.0" ?>
<config>
  <game>
    <title>Space Adventure</title>
    <player>
      <health>100</health>
      <speed>5</speed>
    </player>
    <enemies>
      <item>
        <type>alien</type>
        <health>50</health>
      </item>
      <item>
        <type>robot</type>
        <health>75</health>
      </item>
    </enemies>
  </game>
</config>
```

**Входной файл** `game.conf`:
```conf
let BASE_HEALTH = 100;

$[
    game: $[
        title: q(Space Adventure),
        player: $[
            health: |BASE_HEALTH|,
            speed: 5
        ],
        enemies: {
            $[type: q(alien), health: 50],
            $[type: q(robot), health: 75]
        }
    ]
]
```
